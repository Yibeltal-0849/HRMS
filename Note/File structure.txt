📁 Folder Definitions (Root)
e2e/: End-to-end test setup using Protractor.
node_modules/: Installed dependencies (auto-generated by npm).
src/: Main source code of the application.

📄 File Definitions (Root)
angular.json: Configuration for Angular CLI build and project settings.
package.json: Lists npm dependencies and scripts.
tsconfig.json: TypeScript compiler configuration.
tslint.json: Linting rules for code quality (used before ESLint).
.editorconfig: code formatting rules — like spaces, tabs, and empty lines.
README.md: Project overview or usage instructions
tsconfig.app.json:Tells Angular CLI which files to compile for the main app
tsconfig.json:Base configuration for compiling TypeScript.

📁 Folder Definitions (src/)
app/: Main application folder (contains components, modules, services).
assets/: Static files like images, fonts, icons.
environments/: Environment-specific configurations (dev/prod).

📄 File Definitions (src/)
app-routing.module.ts:controls navigation between different pages (components) of your app
app.component.ts: Root component logic (TypeScript).
app.component.html: Template for root component (HTML).
app.component.scss: SCSS for component styles.
app.module.ts: Root Angular module declaring components/services.
favicon.ico: Browser tab icon.
index.html: Main HTML file that loads the Angular app.
main.ts: Entry point to bootstrap the Angular app.
polyfills.ts: Compatibility code for older browsers.
styles.scss: Global styles in SCSS.
main.ts:It starts the app by loading the root module (AppModule) and runs it in the browser.
